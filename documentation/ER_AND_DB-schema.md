## ER diagram & DB schema 📊

## ER diagram
- is a conceptual model
- shows entities and relationships
- focuses on domain understanding
- technology-agnostic

## DB schema
- is the physical implementation
- is tables, columns, types and constraints
- fouses on how the database is actually built

While an Entity-Relationship diagram showcase the idea, the Database schema show the actual implementation.

Model domain -> Design DB from domain -> design REST from DB -> design flow from REST -> validate flow via state machine
